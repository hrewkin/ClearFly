# План реализации: Система автоматизации авиакомпании «Чистое Небо» (ИС-ЧН-2025)

## 1. Описание задачи и целевое назначение
Реализация комплексной информационной системы для автоматизации процессов бронирования, продаж, обслуживания пассажиров, управления внештатными ситуациями и трекинга багажа авиакомпании «Чистое Небо».
Система будет построена на базе микросервисной архитектуры с использованием языков программирования Go (v1.25+) и Java (v1.23+ со Spring), а также с фронтендом на React.js.

### Основные цели:
- Создание центрального реестра бронирований.
- Разработка веб-интерфейса (сайта) для личного кабинета пассажира.
- Внедрение модулей управления внештатными ситуациями и трекинга багажа.
- Оптимизация управленческой отчетности («почти онлайн»).

---

## 2. Архитектура и инфраструктура

Для обеспечения работы всех компонентов и их независимого масштабирования используется контейнеризация Docker. Взаимодействие сервисов описывается в `docker-compose.yml`.

### 2.1 Инфраструктурные сервисы (БД и Брокеры сообщений)
- **PostgreSQL (v13+)**: Основная реляционная система управления базами данных (СУБД). Хранит информацию о бронированиях (PNR), транзакциях, профилях пользователей и багаже. Выбрана за надежность и поддержку транзакционной блокировки мест (ACID).
- **Redis (v6.0+)**: Используется преимущественно для кэширования сессий, справочников (например, списков аэропортов и тарифов), а также блокировок распределенных транзакций.
- **RabbitMQ**: Выделенный брокер сообщений (Message Broker). Если Redis решает задачи кэширования, то RabbitMQ обеспечивает гарантированную и отказоустойчивую асинхронную передачу событий между микросервисами. Примеры: события сканирования багажа (RFID), команды на массовую рассылку уведомлений при задержках рейсов.

### 2.2 Микросервисы (Бэкенд)
- `gateway` — центральный API-шлюз для маршрутизации трафика к сервисам.
- `booking` (написан на Go) — модуль бронирования и управления инвентарем (работает с PostgreSQL).
- `passenger` (Java/Go) — сервис профилей и личного кабинета.
- `incident` (Java/Go) — модуль управления внештатными ситуациями. Реагирует на сообщения из RabbitMQ-очередей.
- `baggage` (Java/Go) — сервис трекинга багажа.
- `analytics` (Java/Go) — подсистема управления доходностью и аналитики.

### 2.3 Веб-интерфейс
- `webui`: Фронтенд-часть системы. SPA веб-приложение на базе **React.js**. Обеспечивает личный кабинет пассажира, поиск билетов, регистрацию на рейс и отображение статусов багажа. Адаптировано под десктопные и мобильные браузеры.

## 3. Обеспечение качества кода, тестирование и деплой

> [!IMPORTANT]  
> Перед внедрением новой функциональности должен быть пройден цикл проверок: статический анализ (линтеры) и авто-тесты.

### 3.1 Линтеры (Golang)
- Для всех Go-сервисов (например, `booking`) будет настроен конфиг `.golangci.yml`.
- Линтер гарантирует соблюдение стандартов написания кода, отсутствие утечек памяти, неиспользуемых переменных и потенциальных race-conditions.
- Запуск: `golangci-lint run`. Должен выполняться локально, а также быть встроенным шагом в CI-пайплайн.

### 3.2 Комплексное тестирование
По завершению разработки основного каркаса приложения будут написаны и запущены тесты.
- **Unit-тесты**: Покрытие бизнес-логики (например, алгоритмы расчета стоимости и транзакционной блокировки инвентаря). Для Go используются `testing` и `go test`.
- **Интеграционные тесты**: Проверка связки сервиса с базой данных (PostgreSQL) и брокером (RabbitMQ) с использованием TestContainers.
- Прогон тестов будет автоматизирован (команда `go test ./...` или `mvn test` для Java сервисов) до сборки Docker-образов.
