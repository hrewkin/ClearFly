# ClearFly — "Чистое небо"

Демо-стенд АС «Чистое небо»: 8 Go-микросервисов + React UI, общаются по HTTP и через RabbitMQ. Готов к 2-минутной презентации: бронирование по карте мест, публикация инцидентов, фан-аут уведомлений по PUSH/SMS/EMAIL, live-аналитика загрузки рейсов.

## Быстрый старт

Нужен **Docker** и **Docker Compose**. Порты 3000, 8080, 5432, 5672, 15672 должны быть свободны.

```bash
git clone https://github.com/hrewkin/ClearFly.git
cd ClearFly
make up          # docker compose up -d --build
```

Первый запуск — 2–3 минуты (собираются 8 Go-сервисов и фронт). После этого открывайте:

| URL | Что там |
| --- | --- |
| http://localhost:3000 | UI (React) |
| http://localhost:8080 | API-шлюз |
| http://localhost:15672 | RabbitMQ Management (guest / guest) |

### Засеять демо-данные

Сразу после `make up` в системе есть 5 рейсов и пара пассажиров. Чтобы показать всё сразу — пассажиров с разными тирами лояльности, бронирования с разными коэффициентами загрузки, багаж на всех стадиях и отложенный рейс — запустите сидер:

```bash
make demo
```

Или одной командой с чистой БД:

```bash
make reseed      # = down -v + up + seed
```

Скрипт `scripts/seed_demo.py` использует только stdlib Python 3, ничего ставить не нужно. Параметры:

```bash
python3 scripts/seed_demo.py --base-url http://localhost:8080 \
                             --passengers 30 \
                             --seed 1337
```

После сидинга:
- **Аналитика** показывает рейсы с load factor ~20% / ~65% / ~95% — все три тарифных коридора (×1.0 / ×1.2 / ×1.5).
- **Багаж** содержит бирки на всех стадиях от «Сдан» до «Выгружен».
- **Уведомления** уже не пусты (инцидент `FLIGHT_DELAYED` разлетелся по каналам).

### Полезные команды

```bash
make ps          # статус контейнеров
make logs        # tail -f по всем сервисам
make down        # остановить стек (данные сохранятся)
make reset       # остановить и стереть БД
```

## Что показать за 2 минуты

1. **Главная** — KPI-карточки, ближайшие рейсы, лента событий, три архитектурные фичи (атомарное распределение мест, RabbitMQ, динамические тарифы).
2. **Поиск рейсов → Бронирование** — выбор кресла на схеме с разделением бизнес/эконом, мгновенная выдача посадочного талона с PNR.
3. **Операции** — публикация инцидента (задержка / отмена / смена выхода), зелёный баннер подтверждает отправку в шину.
4. **Уведомления** — автообновление, фан-аут по PUSH / SMS / EMAIL, цветовая кодировка по типу события.
5. **Аналитика** — load factor, gauge, подсветка активного правила ценообразования.
6. **Багаж** — 6-стадийный таймлайн, кнопка «Следующий скан» двигает бирку вперёд.
7. **Профиль** — лояльность, питание, особые потребности в один клик.

## Структура

```
services/
  gateway/         # API-шлюз (порт 8080)
  webui/           # React + Vite + nginx (порт 3000)
  booking/         # рейсы, места, брони (порт 8081)
  passenger/       # профиль, лояльность, питание (порт 8082)
  incident/        # инциденты → RabbitMQ (порт 8083)
  baggage/         # багажный трекинг (порт 8084)
  analytics/       # load factor + динамические тарифы (порт 8085)
  notification/    # fan-out уведомлений (порт 8086)
docker/
  docker-compose.yml
scripts/
  seed_demo.py     # генератор демо-данных
```

## Разработка

Каждый Go-сервис самостоятелен (`go build ./...` в своём каталоге). Фронт собирается в `services/webui/` (`npm install && npm run build`). Pre-commit / lint-правила описаны в `.editorconfig` и `services/webui/.eslintrc.cjs`.
