# План доработки ClearFly — что осталось дописать

> Создан: 26 апреля 2026 г.  
> Источник: `Требования_АС_Чистое_Небо_ласт_апдейт_на_паре.docx`  
> Текущее покрытие требований: ~25-30%

---

## 🔴 Критично (4 задачи) — без них система не работает

### 1. Модель рейсов (Flights)
- Таблица `flights`: id, flight_number, origin, destination, departure_time, arrival_time, aircraft_type, total_seats, available_seats, gate, status
- CRUD + поиск рейсов по маршруту и дате
- Страница поиска рейсов в WebUI
- **Сервис:** booking | **Объём:** Большой

### 2. Инвентарь мест (Seats)
- Таблица `seats`: id, flight_id, seat_number, class (ECONOMY/BUSINESS), status (AVAILABLE/BLOCKED/BOOKED), booking_id
- Блокировка мест при бронировании через `SELECT ... FOR UPDATE` (транзакционная)
- **Сервис:** booking | **Объём:** Средний

### 3. Расширение бронирования (PNR, цена, оплата)
- Новые поля в `bookings`: pnr_code, seat_id, price, currency, payment_status (PENDING/PAID/REFUNDED), created_at
- Генерация PNR-кода (6 символов, напр. "XKF3LM")
- **Сервис:** booking | **Объём:** Средний

### 4. Сервис уведомлений (Notification)
- Новый микросервис `notification`
- Таблица `notifications`: id, passenger_id, type, channel (PUSH/SMS/EMAIL), title, content, sent_at, read
- RabbitMQ consumer — слушает очередь `notification_events`
- HTTP API — `GET /notifications/:passenger_id`
- Incident при обработке инцидента публикует в очередь `notification_events`
- **Сервис:** НОВЫЙ notification | **Объём:** Большой

---

## 🟡 Важно (5 задач) — значительно улучшает соответствие

### 5. Расширение пассажира
- Новые поля: `loyalty_status`, `meal_preference`, `special_needs`
- **Сервис:** passenger | **Объём:** Маленький

### 6. Привязка багажа к рейсу
- Новое поле `flight_id` в модели и таблице `baggage_tracking`
- **Сервис:** baggage | **Объём:** Маленький

### 7. Расширение incident
- Новый тип `GATE_CHANGED`
- Публикация события в очередь `notification_events` для notification-сервиса
- Заглушка для перебронирования
- **Сервис:** incident | **Объём:** Средний

### 8. Модель тарифов
- Таблица `tariffs`: id, flight_id, class, base_price, currency
- Расчёт цены при бронировании по тарифу + коэффициент загрузки
- **Сервис:** booking | **Объём:** Средний

### 9. Новые страницы WebUI
- Страница «Поиск рейсов» (откуда/куда/дата → результаты с ценами)
- Страница «Бронирование» (выбор места, данные пассажира, подтверждение)
- Страница «Уведомления» (список уведомлений пассажира)
- **Сервис:** webui | **Объём:** Большой

## 🟢 Желательно (3 задачи) — полнота системы

### 10. RBAC + JWT аутентификация
- JWT-middleware в gateway
- Таблица `users` (email, password_hash, role)
- Роли: PASSENGER, ADMIN, COMMERCIAL, OPERATIONS, GROUND, CREW
- **Сервис:** gateway | **Объём:** Средний

### 11. Расширение аналитики
- Новые эндпоинты: выручка по рейсу, средний доход на место
- Зависит от тарифов (п.8) и расширенных бронирований (п.3)
- **Сервис:** analytics | **Объём:** Маленький

### 12. Онлайн-регистрация (check-in)
- `POST /bookings/:id/checkin` — меняет статус на CHECKED_IN
- Генерация посадочного талона
- **Сервис:** booking | **Объём:** Маленький

1 → 2 → 3 → 8 → 4 → 7 → 5 → 6 → 9 → 10 → 11 → 12
